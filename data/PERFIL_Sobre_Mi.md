# Perfil Profesional: Fabián Albarracín

## 1. ¿Quién es Fabián Albarracín?
Fabián Albarracín es un **Tecnólogo en Desarrollo de Software y Aplicativos Móviles**, con un fuerte enfoque en backend, modelado de datos e integración de sistemas. 

A nivel de mercado, se define como un **Desarrollador Backend Junior con mentalidad estructural**. No es un arquitecto corporativo tradicional ni un perfil senior, sino un profesional en etapa de consolidación que ya posee la madurez para diseñar modelos de datos estructurados, construir backends funcionales e integrar sistemas externos con impacto operativo real.

No se limita a construir interfaces o funcionalidades aisladas; su instinto es analizar el flujo completo de la operación antes de diseñar una solución.

## 2. Enfoque y Metodología de Trabajo
Fabián aborda el desarrollo de software bajo un patrón claro y pragmático:
1.  **Entender:** Comprender el problema real y el dolor del negocio.
2.  **Mapear:** Trazar el flujo operativo completo antes de tocar el código.
3.  **Modelar:** Diseñar correctamente la estructura y el modelo de datos.
4.  **Construir:** Ejecutar una solución funcional respetando restricciones reales (tiempo, presupuesto, infraestructura).
5.  **Iterar:** Mejorar el sistema de manera incremental y continua.

## 3. Competencias Técnicas y Capacidades
Su perfil técnico se sostiene en pilares estructurales y un stack de lenguajes versátil:

### Lenguajes de Programación Core
* **Python:** Lenguaje principal para lógica de backend, scripting, creación de APIs (Flask) y procesamiento de eventos (MQTT).
* **Java y C#:** Fundamentos profundos en Programación Orientada a Objetos (POO), herencia, polimorfismo y modularidad, adquiridos durante su formación tecnológica formal.
* **JavaScript:** Dominio del ecosistema web y comprensión de la lógica interna del motor V8 (Scope, Hoisting, Closures).
* **SQL:** Lenguaje base para el modelado, consulta y normalización de bases de datos relacionales.

### Arquitectura y Backend
* Diseño orientado a eventos y arquitecturas desacopladas.
* Separación de capas de datos (RAW vs. Estructurado).
* Persistencia segura y transaccional (ej. SQLite en modo WAL).
* Diseño de endpoints REST para consumo externo.

### Modelado de Datos
* Diseño relacional con normalización hasta 3FN.
* Mantenimiento estricto de la integridad referencial.
* Trazabilidad absoluta mediante identificadores únicos (UID).

### Integraciones y Aceleradores (Low-Code)
* Uso estratégico de **Appian** (Integrations, SAIL Dashboards) y **AppSheet** para soluciones internas y control de estados.
* El uso de herramientas Low-Code forma parte de su estrategia de entrega ágil, pero **no reemplaza** su base en diseño estructural y lógica de programación.
