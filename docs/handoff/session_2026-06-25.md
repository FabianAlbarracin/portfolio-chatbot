## Problema abierto: Alucinaciones en educacion

### Sintoma

El LLM (`llama-8b`) inventa cursos, certificaciones, plataformas y anos que NO estan en
el documento `formacion_academica.md`. Ejemplo: el chatbot dijo que Fabian tiene
"Certificacion en Desarrollo de Aplicaciones Web con Python: Platzi, 2023" y
"Certificacion en Arquitectura de Software: Microsoft Virtual Academy, 2019" —
ninguna existe en el documento real.

### Causa raiz

El system prompt tiene la Regla 1 "CERO ALUCINACIONES" y la Regla 3 "NO INVENTES
TECNOLOGIAS NI PROYECTOS", pero el modelo las ignora cuando se le pide listar
cursos/certificaciones. Tiende a "rellenar" con nombres de plataformas prestigiosas
(Coursera, edX, Microsoft) y anos para dar una respuesta que suene completa.

### Que entidades estan en riesgo

Cualquier documento con listas es vulnerable:
- **`formacion_academica.md`** (CRITICO): cursos, certificaciones, fechas, plataformas
- **`experiencia_profesional.md`** (ALTO): nombres de empresas, fechas, tecnologias
- **`perfil_personal.md`** (MEDIO): habilidades, idiomas, anos de experiencia
- **Proyectos** (BAJO): el stack tecnologico suele respetarse porque las tecnologias
  aparecen explicitamente en los chunks

### Por que `_check_confidence` no lo detecta

La funcion solo verifica si `entity_names` de los sources aparecen en el texto de la
respuesta. No analiza si el CONTENIDO de la respuesta es fiel al contexto. Cuando el
LLM mezcla datos reales con inventados, la entidad "formacion_academica" aparece en
los sources asi que `confidence` sale `high`.

### Posibles soluciones (a evaluar en proxima sesion)

1. **Reforzar system prompt** — Anadir regla explicita anti-invencion de cursos
2. **Aumentar `k` del retriever** — De 6 a 10 chunks para dar mas contexto real
3. **Verificacion a nivel de tokens** — Extraer entidades nombradas del contexto
   (plataformas, certificaciones) y verificar que la respuesta no mencione ninguna
   fuera de esa lista
4. **Cambiar modelo** — `llama-8b` podria ser mas propenso a alucinar que otros modelos
   disponibles via LiteLLM

## Trampas conocidas

1. **Workers=1 es obligatorio.** ChromaDB usa SQLite. Si aumentas workers, rompe.
2. **El orden del ingest importa.** `ingest.py` primero, luego `POST /admin/refresh`.
3. **No uses `print()`.** El contrato exige `logging` module con niveles.
4. **`data/**/*.md` esta en .gitignore.** No committees informacion personal.
5. **LiteLLM es el proxy.** Las API keys reales NO estan en este repo.
6. **`entity_name` sin `.md`** es la convencion v2.0.
7. **`ALLOWED_ORIGINS` debe configurarse en `.env` de produccion.** El default
   `localhost:5173` solo sirve para desarrollo local. Sin esta variable, el frontend
   en produccion recibe errores de CORS.
8. **`docker compose up -d` sin `--build` no reconstruye la imagen.** Si cambias
   `pyproject.toml` o `Dockerfile`, necesitas `docker compose build --no-cache`
   antes de levantar.
9. **`create_retrieval_chain` NO existe en LangChain 1.2.x.** La implementacion
   real usa LCEL (`itemgetter` + `RunnablePassthrough` + `RunnableLambda`).
   No intentes usar las APIs del CONTRATO que ya no existen.
10. **El LLM alucina cursos y certificaciones.** Ver seccion "Problema abierto" arriba.
