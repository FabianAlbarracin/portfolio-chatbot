## Como ejecutar y testear

```bash
# Arrancar (requiere red externa homelab_proxy_net)
docker compose up -d

# Actualizar conocimiento (ingest + hot-reload)
bash update_knowledge.sh

# Probar el chat
curl -X POST http://localhost:8000/chat \
     -H "Content-Type: application/json" \
     -d '{"question": "Hablame sobre Tradehub", "session_id": "abc123"}'

# Tests unitarios
pytest tests/unit/ -v

# Evaluador de calidad (requiere API corriendo + GROQ_API_KEY en .env)
python tests/evaluator.py

# Healthcheck
curl http://localhost:8000/health
```

## Trampas conocidas

1. **Workers=1 es obligatorio.** ChromaDB usa SQLite. Si aumentas workers, rompe.
2. **El orden del ingest importa.** `ingest.py` primero, luego `POST /admin/refresh`. Si
   solo ejecutas ingest sin hot-reload, el servidor usa vectores viejos en RAM.
3. **No uses `print()`.** El contrato exige `logging` module con niveles.
4. **`data/**/*.md` esta en .gitignore.** No committees informacion personal.
5. **LiteLLM es el proxy.** Las API keys reales de Groq NO estan en este repo.
   Solo `LITELLM_CHATBOT_KEY` (virtual key).
6. **El few-shot del router legacy esta hardcodeado** con nombres de entidades.
   Al migrar a v2.0, esto desaparece — no intentes arreglarlo.
7. **`entity_name` sin `.md`** es la convencion v2.0. Los archivos fisicos tienen
   extension `.md` pero el frontmatter usa el nombre sin extension.

## Decisiones pendientes

- **`translated_query`**: eliminado en v2.0. La busqueda usa la query cruda del usuario.
  Si las pruebas muestran baja precision en retrieval, evaluar reformulacion de query.
- **Qdrant/Milvus**: descartado para v2.0. ChromaDB es suficiente para <100 consultas/dia.
- **Watchdog/auto-ingesta**: descartado. El script manual `update_knowledge.sh` es suficiente
  para 7 documentos.
- **Multi-idioma**: delegado 100% al LLM via system prompt. Si el LLM falla en algun idioma,
  implementar deteccion programatica como fallback en fase 2.
- **Evaluador**: migrar de LLM Judge a metricas RAGAS (Context Relevance, Faithfulness,
  Answer Relevance).
